# Answers

## 1. Purpose of the Study
The project aims to develop and utilize a secured web-based system for processing and dealing with student examination results in institutions of higher learning.

## 2. The Study Aim
The aim of this project is to build a secured online student management and examination results processing system that supports role-based access, result calculation, and reliable data handling.

## 3. Research Aims and Achievement Status

### Aim 1
To analyze the current practices in the tertiary institution regarding the processing of results to identify the functional requirements, deficiencies, and limitations in the current workflow.

**Answer:** Partially achieved. The project implements a structured student management system and demonstrates an understanding of required functions through its dashboard modules and workflow. The evidence for a formal analysis of existing institutional processes is not explicit in the codebase, but the system does address common deficiencies such as manual result handling and lack of centralization.

### Aim 2
To design an online secured system that supports the processing of result data and provides security access based on roles.

**Answer:** Achieved. The project includes separate login and dashboard areas for admin, lecturers, and students, showing a role-based login system and access control. Security features like OTP-based password reset and separate user access are present.

### Aim 3
To design an online system that calculates and stores results without any errors and inefficiency on the part of humans.

**Answer:** Achieved. The project supports semester marks, CGPA calculations, and result storage in a database, reducing manual error and improving efficiency compared to paper-based processes.

### Aim 4
To incorporate the security concept that only authorized persons can input, update, approve, and view the result.

**Answer:** Achieved. The project structure includes admin and staff login flows, and separate dashboard functionality, which indicates that only authorized users can access result entry and management features.

### Aim 5
This system will be validated and verified to work well regarding accuracy, security, reliability, and usability.

**Answer:** Partially achieved. The codebase provides functional modules, role-based access, and result processing, suggesting reliability and usability. Formal validation and verification documentation or testing results are not visible in the repository, but the implemented system architecture supports these objectives.

## 4. Overall Conclusion
The project largely meets the stated objective of building a secured web-based system for student result processing. Core goals like role-based security, result calculation, storage, and user-specific dashboards are implemented. Some research-related goals, such as documented analysis of existing workflows and explicit validation reports, are not clearly documented in the repository.
